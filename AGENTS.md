## Superpowers 필수 정책

- 에이전트가 개발, 이슈 등록, PR 작성·검토 등 저장소 작업을 수행할 때는 반드시 먼저 `using-superpowers`를 적용하고, 작업에 해당하는 Superpowers 스킬을 사용하십시오.
- Superpowers 절차를 거치지 않은 개발, 이슈 등록, PR 작성·검토를 금지합니다.
- 에이전트가 작성하는 모든 GitHub 이슈 본문과 PR 본문에는 Superpowers 사용 인증 표시 `⚡️`를 반드시 포함하십시오.
- 이슈 또는 PR을 제출하기 직전에 본문에 `⚡️`가 포함되었는지 검증하십시오.
- 브랜치, 이슈, PR, 커밋 및 배포 절차는 `CONTRIBUTING.md`를 따르십시오.
